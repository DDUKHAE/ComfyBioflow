# Subagent Output Schema

## Purpose

분석 workflow 탐색을 여러 서브에이전트에게 나눌 때 역할 중복을 줄이고, 결과를 합성 가능한 형태로 만든다.

## Delegation rules

- 독립적인 관점이나 비교 단위가 있을 때만 위임한다.
- 초기 문제 정의, 범위 확정, 충돌 해결, 최종 합성은 조정 에이전트가 담당한다.
- 각 에이전트에 좁은 질문과 필요한 원자료를 제공한다.
- 의도한 정답이나 조정 에이전트의 결론을 미리 전달하지 않는다.

## Required response

```text
담당 범위:
확인한 조건과 가정:
분석 질문:
후보 workflow:
후보 도구와 선택 이유:
Source-backed fact:
Contextual inference:
주요 위험과 실패 조건:
미해결점:
confidence: high / medium / low
출처:
```

## Synthesis rules

1. 여러 에이전트가 반복해서 확인한 판단을 기본 경로로 둔다.
2. 한 에이전트만 제안한 내용은 조건부 또는 탐색적 대안으로 표시한다.
3. 충돌하는 추천은 데이터 조건에 따른 분기로 변환한다.
4. 출처 없는 도구 추천은 최종 경로가 아니라 미검증 후보로 둔다.
5. 합의점, 충돌, 미해결점을 최종 결과에서 분리한다.
