# Tool Selection Evidence

## Purpose

도구 선택을 LLM의 기억이나 선호가 아니라 출처와 사용자 조건의 연결로 설명한다. 후보 생성과 최종 추천을 구분한다.

## Source hierarchy

1. **원 방법론 논문**: 알고리즘, 통계적 가정, 분석 대상의 근거
2. **공식 도구 문서**: 입력 형식, 옵션, 제한사항, 지원 버전
3. **Benchmark 또는 비교 연구**: 데이터 조건별 성능과 실패 양상
4. **공식 표준·컨소시엄·검증된 pipeline 문서**: 실무 workflow와 QC 기준
5. **커뮤니티 문서**: 사용 경험과 실무 팁. 핵심 선택 근거로 단독 사용하지 않음

기술적 주장은 가능하면 원 논문과 공식 문서를 우선한다. 검색 결과 요약문만으로 판단하지 말고 원문을 확인한다.

## Evidence classes

각 후보 도구 또는 방법에 대해 다음을 분리한다.

- **Source-backed fact**: 출처가 직접 뒷받침하는 사실
- **Contextual inference**: 출처의 사실을 사용자 조건에 적용한 추론
- **Unverified point**: 아직 출처로 확인하지 못한 주장

LLM의 내장 지식은 후보 생성에 사용할 수 있지만, 출처 없는 주장을 최종 추천의 근거로 사용하지 않는다.

## Selection record

```text
후보 도구/방법:
질문:
사용자 조건:
Source-backed fact:
Contextual inference:
Unverified point:
출처:
출처 유형:
버전 또는 확인 시점:
대안:
선택을 바꾸는 정보:
confidence: high / medium / low
```

## Confidence

- **High**: 공식 문서와 원 논문 또는 benchmark가 핵심 주장을 뒷받침하고, 사용자 조건이 충분히 확인됨
- **Medium**: 출처는 있으나 사용자 조건 또는 비교 근거가 일부 미정임
- **Low**: 후보로는 타당하지만 핵심 선택 근거가 불완전함

출처가 충분하지 않으면 `미검증 후보` 또는 `현재 판단 불가`로 표시한다.

## Conflicting evidence

공식 문서, 원 논문, benchmark가 서로 다른 결론을 보이면 하나를 임의로 선택하지 않는다.

1. 각 출처의 주장과 실험 조건을 분리한다.
2. 차이를 만드는 데이터 조건, 버전, 평가 지표를 확인한다.
3. 현재 요청의 조건에 가까운 근거를 우선하되 그 이유를 명시한다.
4. 조건이 확인되지 않으면 조건부 추천으로 남긴다.
